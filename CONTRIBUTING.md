# Contributing to SpectrumQ

Thank you for your interest in contributing to SpectrumQ.

## Development Philosophy

SpectrumQ prioritizes:
- Cryptographic correctness
- Clear protocol design
- Security-first engineering
- Transparent review processes

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Ensure all cryptographic or protocol changes are documented
4. Include threat-model considerations for any new feature
5. Submit a pull request with clear rationale

## Code Standards

- No unreviewed cryptographic primitives
- Document all assumptions explicitly
- Maintain modular architecture
- Avoid premature optimization

## Security Considerations

All contributions must consider:
- Metadata leakage
- Forward secrecy implications
- Attack surface expansion

## Review Process

All PRs will be reviewed for:
- Security implications
- Protocol correctness
- Architectural consistency
