# Security Policy

## Supported Versions

SpectrumQ is currently in early development. No version is considered production-ready.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly by opening a private security advisory on GitHub.

Do NOT publicly disclose exploits before a fix is available.

We take all security reports seriously and will respond as quickly as possible.

## Scope

This includes:
- Cryptographic design flaws
- Protocol weaknesses
- Implementation vulnerabilities
- Metadata leakage vectors
- Transport layer attacks

## Principles

- Security through design review, not obscurity
- Public accountability for cryptographic decisions
- Independent review encouraged
